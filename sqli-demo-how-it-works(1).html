How SQL Injection Works

Understanding SQL Injection

SQL Injection is a code injection technique that exploits security vulnerabilities in an application's software by inserting malicious SQL statements into entry fields for execution.

Common SQL Injection Techniques

Authentication Bypass
Example: ' OR '1'='1

When this is injected into a login form:

SELECT * FROM users WHERE username = '' OR '1'='1' AND password = 'anything'

The OR '1'='1' always evaluates to true, causing the query to return all users, typically logging in as the first user in the database.

UNION-based Attacks
Example: 1 UNION SELECT username, password FROM users

This technique allows attackers to extract data from different database tables:

SELECT field1, field2 FROM table1 WHERE id = 1 UNION SELECT username, password FROM users

This causes the query to return data from the users table, potentially exposing sensitive information.

Extracting Database Information
Example: 1 ORDER BY 5

This technique helps attackers determine the number of columns in a table:

SELECT * FROM products ORDER BY 5

If the query executes without error, it means the table has at least 5 columns.

Boolean-based Blind Injection
Example: 1 AND (SELECT COUNT(*) FROM users WHERE username LIKE 'a%') > 0

Used when no direct output is visible, but the application behaves differently based on whether the SQL statement is true or false.

Prevention Techniques

Parameterized Queries
Safe Example:
query = "SELECT * FROM users WHERE username = ? AND password = ?"
cursor.execute(query, (username, password))

Parameters are sent separately from the SQL statement, preventing injection attacks.

Input Validation
Example:
// Check if input is numeric
if (!isNumeric(userInput)) {
    return "Invalid input";
}

Validate and sanitize all user inputs before using them in SQL queries.

Remember: This application is for educational purposes only. Always use parameterized queries and proper input validation in real applications.
